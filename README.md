# F.CSM306 Project

Энэхүү төсөл нь `merge sort` алгоритмын гүйцэтгэлийг дараах 3 CPU хувилбараар харьцуулан хэмжих зорилготой:

- `serial`
- `std::thread`
- `OpenMP`

Туршилтын үр дүнг `csv/output.csv` файлд хадгалж, дараа нь Python скриптээр уншин график болгон дүрсэлнэ.

## Төслийн зорилго

Энэхүү ажлын үндсэн зорилго нь:

- `merge sort` алгоритмын энгийн дараалсан хувилбарыг хэрэгжүүлэх
- `std::thread` ашиглан параллель хувилбар гаргах
- `OpenMP` ашиглан ижил санааг өөр орчинд хэрэгжүүлэх
- ижил оролт дээр 3 аргын хугацаа, speedup, achievable performance-ийг харьцуулах

## Ашигласан аргачлал

Төсөлд `merge sort` алгоритмын рекурсив бүтэц ашигласан. Параллель хувилбаруудад:

- массивыг зүүн, баруун хоёр хэсэгт хуваана
- хэсэг тус бүрийг рекурсив байдлаар эрэмбэлнэ
- дараа нь `co-rank` санаа ашиглан merge үе шатыг параллельчилна

`std::thread` болон `OpenMP` хоёрын ялгаа нь параллель ажлыг зохион байгуулж буй орчин юм. Харин үндсэн merge-ийн логик нь ижил чиглэлтэй.

## Хавтас, файлуудын бүтэц

- `src/main.cpp`
  Бенчмарк ажиллуулж, хэмжилтүүдийг `csv/output.csv` файлд бичнэ.
- `src/tasksys.cpp`
  `serial`, `std::thread`, `OpenMP` хувилбаруудын хэрэгжүүлэлт.
- `src/tasksys.h`
  Функц болон классуудын зарлал.
- `src/itasksys.h`
  Нийтлэг интерфэйс.
- `csv/output.csv`
  Бенчмаркийн түүхий үр дүн.
- `visualization/main.py`
  CSV файлыг уншиж дундаж утгуудыг бодон график үүсгэнэ.
- `output/`
  Үүсгэсэн зургууд хадгалагдах хавтас.

## Туршилтын тохиргоо

Одоогийн байдлаар `main.cpp` дээр:

- thread-ийн тоо: `20`
- нийт давталтын тоо: `10`
- массивын хэмжээнүүд:
  - `10000`
  - `100000`
  - `1000000`

Нэг `run_id` бүр дээр:

- шинэ санамсаргүй массив үүсгэнэ
- ижил массивыг 3 аргаар тус тус эрэмбэлнэ
- үр дүн зөв эсэхийг шалгана
- хэмжилтүүдийг CSV файлд хадгална

Ингэснээр нэг ижил оролт дээр аргуудыг шударгаар харьцуулах боломжтой.

## CSV багануудын тайлбар

`csv/output.csv` файлд дараах баганууд хадгалагдана:

- `method`
  Ашигласан арга. `serial`, `threads`, `openmp`.
- `input_size`
  Эрэмбэлэх массивын хэмжээ.
- `num_threads`
  Ашигласан thread-ийн тоо.
- `run_id`
  Давталтын дугаар.
- `execution_time_ms`
  Тухайн аргын ажилласан хугацаа миллисекундээр.
- `data_transfer_time`
  CPU хувилбаруудад `0`. GPU хувилбар нэмбэл ашиглаж болно.
- `data_transferred_bytes`
  CPU хувилбаруудад `0`.
- `total_operations`
  `merge sort`-ийн ажлын хэмжээг ойролцоолсон тооцоолол.
- `achievable_performance`
  `total_operations / execution_time_seconds` томьёогоор бодсон throughput.

Анхаарах зүйл:

- `speedup` нь CSV файлд шууд хадгалагдахгүй
- `speedup`-ийг Python скрипт дундаж хугацаанаас бодно

## Python скрипт юу хийдэг вэ

`visualization/main.py` нь:

- `csv/output.csv` файлыг уншина
- нэг `method` ба `input_size` дээрх олон `run_id`-ийн дундаж хугацааг бодно
- `serial` аргыг суурь болгож speedup тооцно
- дараах 3 графикийг үүсгэнэ

### 1. Execution Time

- `x` тэнхлэг: `input_size`
- `y` тэнхлэг: `execution_time_ms`

Энд бага утга нь сайн үзүүлэлт.

### 2. Speedup

- `x` тэнхлэг: `input_size`
- `y` тэнхлэг: `speedup`

Томьёо:

```text
speedup = average_serial_time / average_method_time
```

Энд их утга нь сайн үзүүлэлт.

### 3. Achievable Performance

- `x` тэнхлэг: `input_size`
- `y` тэнхлэг: `achievable_performance`

Энэ нь тухайн арга нэгж хугацаанд хэчнээн хэмжээний ажлыг гүйцэтгэж байгааг харуулна.

## Үр дүнг хэрхэн тайлбарлах вэ

Графикуудыг харахдаа дараах зүйлсийг анхаарах нь зүйтэй:

- `execution time` бага байх тусам сайн
- `speedup` өндөр байх тусам параллельжилт илүү үр дүнтэй
- `achievable performance` өндөр байх тусам throughput сайн
- жижиг `input_size` дээр параллель overhead их байж болох тул `serial` аргын эсрэг давуу тал бага гарч болно
- том `input_size` дээр `std::thread` болон `OpenMP` илүү сайн үр дүн үзүүлэх хандлагатай

## Одоогийн төслийн хүрээ

Энэ репозиторт одоогоор CPU дээрх 3 хувилбарын харьцуулалт төвлөрсөн байгаа:

- `serial`
- `std::thread`
- `OpenMP`

GPU буюу CUDA хувилбар нь тусдаа орчинд хөгжүүлж, туршиж болохоор салангид авч үзэж болно.

## Цааш өргөтгөж болох санаанууд

- `efficiency = speedup / num_threads` график нэмэх
- стандарт хазайлт эсвэл error bar дүрслэх
- илүү олон `input_size` турших
- өөр thread-ийн тоонууд дээр нэмэлт туршилт хийх
- CPU ба GPU үр дүнг тусад нь нэгтгэн эцсийн харьцуулалт хийх

## Дүгнэлт

Энэхүү төсөл нь `merge sort` алгоритмын гүйцэтгэлийг параллель орчинд хэрхэн сайжруулж болохыг практик байдлаар харуулж байна. `serial`, `std::thread`, `OpenMP` гэсэн гурван хувилбарыг ижил өгөгдөл дээр олон давталтаар ажиллуулж, түүхий хэмжилтээс график гарган харьцуулж буй нь туршилтын үр дүнг ойлгомжтой, тайлбарлахад хялбар болгож байна.
